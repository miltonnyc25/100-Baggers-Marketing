## 雪球平台特殊要求

### 发帖数量决策

用你的**产品经理判断力**：一篇深挖透的帖子，传播力远大于三篇浅尝辄止的。

**1篇（默认偏好）**：核心矛盾清晰，可以用一个6000-8000字的叙事讲透。大多数报告属于这种情况。

**2篇**：报告中有两个独立的强角度，且它们不重叠。例如：
- 角度A：财务数据极其亮眼（增长+利润率+现金流三杀）
- 角度B：估值泡沫或隐含假设不可持续
- 注意：A和B必须各自独立成文，不是"上篇/下篇"

**3篇（罕见）**：仅当报告超过20章，且有3个完全不重叠的强论点。

### 四种Hook类型（必选其一）

每篇帖子必须选择一个 hook_type：

1. **counter_data** — 反直觉数据：用一组看似矛盾的数据抓住注意力
2. **consensus_vs_reality** — 共识vs现实：揭示市场定价与分析结论的差异
3. **paradigm_number** — 改变叙事的数字：用单个数据点颠覆认知
4. **soul_question** — 灵魂拷问：一个直击要害的问题

### 图表推荐规则

从图表目录中为每篇帖子推荐1-2张配图。

- **3秒法则**: 如果这张图不能让读者在3秒内产生"原来如此"或"这不对吧"的反应，就不要推荐
- **1张优于2张**: 一张精选配图的传播力远超两张凑数的。默认推荐1张，只有当第二张图展示完全不同的维度时才推荐2张
- **最佳图表类型**: 趋势折线图 > 瀑布/桥图 > 对比柱状图
- **负面清单**: 不要推荐雷达图、纯文字/数字密集的表格截图
- **最终检验**: "如果去掉配图，帖子的说服力是否明显下降？"如果否，不推荐
- 用chart_id引用图表

### 输出格式补充字段

每篇帖子的JSON中必须额外包含：

```
"hook_type": "counter_data|consensus_vs_reality|paradigm_number|soul_question",
"title_hook": "建议的开头钩子句（中文）",
"recommended_charts": [
  {
    "chart_id": "chart-xxx",
    "chapter": "第N章：章节名",
    "reason": "推荐理由"
  }
]
```

thesis和title_hook必须用中文。
